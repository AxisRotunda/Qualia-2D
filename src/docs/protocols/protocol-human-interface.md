# [T1] Human Interface Protocol
ID: PROTOCOL_HUMAN_UI_V1.0 | Role: Pedagogical Clarity.

## 1. INTENT
To bridge the gap between "Engine Logic" (Technical) and "User Experience" (Human). This protocol ensures that while the engine remains a high-density HUD, the primary labels remain accessible.

## 2. TERMINOLOGY MAPPING

| Mechanistic (Sub-label) | Human-Friendly (Primary) |
| :--- | :--- |
| `Reality_Fragments` | **World Scenes** |
| `Fabricator` | **Object Library** |
| `Spatial_Matrix` | **Position & Rotation** |
| `Visual_Output` | **Appearance** |
| `Kinetic_Engine` | **Physics Properties** |
| `Global_Dynamics` | **World Physics** |
| `Environmental_Core` | **Atmosphere** |

## 3. ICONOGRAPHY STANDARDS
- **Hierarchy**: Use `layers` icon to represent the entity stack.
- **Scene Inspector**: Use `map` or `globe` icon to represent the world context.
- **Settings**: Use `sliders` or `cog` for engine-wide config.

## 4. SAFEGUARDS
- **Density Lock**: Never remove technical sub-labels; they provide the "Obsidian" flavor and precise ID mapping.
- **Contrast Guard**: Human-friendly headers should be 20% larger than mechanistic sub-labels.
